### ‚úÖ What is a fitting model?

A **fitting model** is any rule, condition, or heuristic the model proposes to classify data points into classes.  
Even if the rule is later rejected, restated, or redundant ‚Äî it must be included.

1. "order" ‚Äî the position in which the model proposed it (starting from 0)
2. "description" ‚Äî a concise, human-readable description of the rule
3. "rule" ‚Äî a dictionary where each class label (e.g., "0", "1") maps to a list of logic expressions

5. "function" ‚Äî a Python function implementing the rule with signature `def model(x, y, data)`
    - All constants must be hardcoded in the function.
    - The function must be fully executable.
6. "extracted_parameter" ‚Äî All constant parameters used in the function (excluding x, y, data). If none, use [].
7. "rule_original_text" ‚Äî the exact original text where the model proposes this rule
8. "validation_original_text" ‚Äî the original text where the model validates this rule, or leave "" if not explicitly validated
